# RAG 文档问答系统

这是一个面向企业制度、流程规范和办公文档的检索增强生成（RAG）项目，支持将本地文档构建为知识库，并对用户问题进行检索、召回、重排与答案生成。

## 项目定位

这个子项目的目标是解决“企业内部文档很多，但难以快速查找和问答”的问题。系统会把制度文档、流程文档、规范文档等内容接入知识库，用户可以直接通过自然语言提问，例如：

- 差旅报销标准是什么？
- 请假审批流程怎么走？
- 合同审批需要哪些材料？
- 发票管理规范有哪些要求？

系统会先检索相关文档片段，再结合上下文生成回答，并返回引用来源。

## 核心能力

- 本地文档导入与预处理
- 文本清洗与切分
- 向量化与向量检索
- 检索结果重排
- RAG 问答生成
- 离线评估数据集与效果评测
- 简单 Web 页面展示
- API 服务化调用

## 技术栈

- `FastAPI`
- `ChromaDB` / 向量存储
- `Embedding` 模型
- `RAG` 检索增强生成流程
- `Pydantic`
- `前端静态页面`

## 目录结构说明

- `main.py`：程序入口，负责启动服务
- `api.py`：对外 API 接口定义
- `rag_system.py`：RAG 主流程编排
- `document_processor.py`：文档加载、切分、清洗
- `retrieval.py`：召回与检索逻辑
- `vector_store.py`：向量库封装
- `embeddings.py`：Embedding 相关封装
- `models.py`：数据模型定义
- `config.py`：配置管理
- `text_cleaning.py`：文本清洗规则
- `utils.py`：通用工具方法
- `eval_offline.py`：离线评测脚本
- `web/index.html`：前端展示页面
- `sample_docs/`：示例文档数据
- `eval_dataset*.json`：评测数据集样例

## 工作流程概览

```text
用户提问
  -> 文档预处理
  -> 文本切分
  -> 向量化
  -> 召回相关片段
  -> 重排排序
  -> 生成答案
  -> 返回引用来源
```

## 本地运行

1. 安装依赖

```bash
pip install -r requirements.txt
```

2. 配置环境变量

复制 `.env.example` 为 `.env`，并按需填写配置。

3. 启动服务

```bash
uvicorn main:app --reload --port 8000
```

4. 访问页面

```text
http://127.0.0.1:8000/
```

## 适合面试的表达方式

你可以把这个项目概括成：

> 这是一个企业知识库 RAG 项目，核心是把制度、流程、规范等文档做成可检索知识库，再通过向量召回、重排和答案生成，为用户提供精准的问答服务。项目还支持 Web 展示、API 服务和离线评测，方便验证检索效果和回答质量。

## 设计亮点

- **文档到知识库的完整链路**：从清洗、切分到召回、生成一条链路打通
- **可扩展**：后续可接入更多文档类型和更强的模型
- **可评测**：内置离线评测数据与脚本，方便量化效果
- **易展示**：同时提供 Web 页面和 API 接口
